摘要

本规范定义了万维网（World Wide Web）的核心语言：超文本标记语言（HTML）的第五次重要修订。当前版本引入了新特性来帮助网页应用（Web application）的作者，引入了基于流行的创作实践的新元素，并且特别注意了给用户代理（user agents）定义清楚的、一致的标准从而提升互用性（interoperability）。

文档状态

本节描述了本文档发布时的状态。其他文档可能会替代本文档。当前的W3C刊物以及最新的技术报告的列表可以在W3C技术报告索引http://www.w3.org/TR/中找到。

W3CHTML工作小组是W3C负责这份规范进度的工作小组。本规范是2016年10月28日的推荐标准。

如果你想以一种W3C可以跟踪的方式给本文档评论，请通过我们的公共bug数据库提交评论（创建bug）。如果你不能这么做，你也可以通过给public-html@w3.org发邮件反馈（订阅，归档），会有安排把这些反馈转移到我们的公共bug数据库中。欢迎各种反馈。

本规范有一份不断更新的测试集（test suite）作为 WebPlatform Tests 项目的一部分。请参看工作组的实现（implementation）报告。

本规范的进展工作进行在http://www.w3.org/TR/html/。HTML5推荐标准代表了HTML发展的一个里程碑，但这还远不是路途的终点，还有许多改进已经蓄势待发。将来的版本可能不会再作为一个独立的规范发布，而是作为一套（set）更小的模块（module）。不管是否如此，http://www.w3.org/TR/html/ 都会被作为全部HTML技术的一个入口来维护。

本规范的扩充特别地通过扩充规范（extension specifications）进行，在这里会商讨什么新特性将被评估。

本规范文本的大部分内容也可以在WHATWG HTML 现存标准找到，这基于一份允许再次使用规范文本的证书。

本文档已被W3C成员、软件开发者和其他W3C组及兴趣小组评审过，并且也已被W3C推荐标准的总监核准。这是一份稳定的文档并且可能被用作参考资料或从其他文档中引用。W3C在制定这份推荐标准的过程中起到了为本规范引起关注和促进其广泛部署（deployment）的作用。这增强了万维网的功能性（functionality）和互用性（interoperability）。

本规范在WHATWG的工作也已完成。W3C HTML工作组积极地追求HTML规范和网页超文本应用技术工作小组现存标准（WHATWG living standard）在W3C HTML 工作组许可的范围之内的一致。有多种方式可以在WHATWG追随这项工作：

  • （译注：通过）提交观察者的邮件列表（完全的根源差异（complete source diffs））：http://lists.whatwg.org/listinfo.cgi/commit-watchers-whatwg.org
  • 标准的差异（ unified diffs）的注释版摘要：http://html5.org/tools/web-apps-tracker
  • Raw Subversion 接口：svn checkout http://svn.whatwg.org/webapps/

以下的编辑修改自从建议推荐标准（Proposed Recommendation）时就已制定：

  • 分类本规范更新的发布方式。
  • application/xhtml+xml 的IANA登记(IANA registration，译注：The Internet Assigned Numbers Authority，互联网数字分配机构)在建议推荐标准时期就已被接受，并给相应部分带来了一个小规模的更新。
  • 当“解码器错误（Decoder error）”事实上是编码相关问题时，它却被错误地列入了URL（译注：统一资源定位器）相关定义
  • “解析URL（Parsed URL）”已经被设置了一个到URL规范的合适映射（mapping）。
  • 改进了一些部分（elements）参考性的摘要。（大部分指向（Mostly pointers）标签省略（tag omission）和ARIA规则（ARIA rules）。）
  • 更新了一些已经看到新的修订的文档的参考 (RFC4281, RFC2313, RFC3490, MPEG-DASH)，并提供了到BECSS（译注：Behavioral Extensions to CSS，https://www.w3.org/TR/becss/）的更稳定的链接。
  • 给URL参考添加了一个相关笔记并制定了一些修改。
  • 进行了一些样式调整和修改了排印错误。
